# kaoshi Spec

`spec.md` 是 kaoshi 的当前产品事实主干。它记录已经确认的产品定位、业务边界、核心概念、用户路径、技术方向和验收标准。

## 1. 产品定位

kaoshi 是一个现代化考试与考试管理平台。

平台服务于学校、机构、团队和个人组织考试的完整流程：建设题库、维护试题、编排试卷、发布考试、考生作答、自动评分、成绩归档和结果查看。

## 2. 当前技术方向

- 后端使用 Java 21 和 Spring Boot 3。
- 数据库使用 MySQL 8。
- ORM 使用 MyBatis-Plus。
- 认证权限使用 Spring Security 和 JWT。
- 缓存、限流和考试运行状态辅助能力使用 Redis。
- 管理端使用 Vue 3、TypeScript、Vite 和 Element Plus。
- 考试端使用 Vue 3 和 TypeScript。
- 部署使用 Docker Compose 组织 MySQL、Redis、后端服务和前端服务。

## 3. 核心用户

- 系统管理员：管理平台配置、用户、角色、权限和全局数据。
- 考务管理员：维护题库、试题、试卷、考试安排和成绩。
- 教师或命题人：创建题目、维护解析、组卷和查看考试结果。
- 考生：查看考试任务、进入考试、作答、提交和查看结果。

## 4. 核心概念

### 4.1 用户与权限

用户是平台身份主体。角色和权限控制管理端能力、考试发布能力、题库维护能力和成绩查看范围。

当前已实现管理员登录、当前用户查询、用户分页搜索、用户新建、用户编辑、用户启用禁用、用户角色分配、角色新建、角色编辑、角色权限分配、角色菜单分配、权限清单和菜单清单。

### 4.2 题库

题库是试题的组织容器。题库支持分类、标签、题型、难度、知识点和归属范围。

当前已实现试题分类、题库列表、新建题库、编辑题库和题库状态维护。

### 4.3 试题

试题是考试内容的基本单位。试题包含题干、题型、选项、答案、解析、分值、难度、标签和附件。

当前已实现单选题、多选题、选项、正确答案、解析、分值、难度、状态和附件元数据。初始化样例包含图片附件和音频附件。

### 4.4 试卷

试卷由试题组成。试卷记录题目顺序、分值快照、总分、限时、说明和组卷规则。

当前已实现试卷分类、试卷列表、新建试卷、编辑试卷、选择试题、题目顺序、分值快照、总分和限时。

### 4.5 考试

考试引用试卷并定义可见范围、开始时间、结束时间、限时、提交规则、成绩规则和发布状态。

当前已实现考试列表、新建考试、编辑考试、引用试卷、开始时间、结束时间、限时、草稿、发布和关闭状态。

### 4.6 作答

作答记录考生每道题的答案、用时、提交状态、评分结果和最终成绩。

当前已实现考试任务列表、开始作答、题目图片/音频/文件附件展示、答案提交、提交锁定和重复提交拒绝。

### 4.7 成绩

成绩是作答提交后的结果归档，包含总分、得分、正确率、用时、题目明细和可查看范围。

当前已实现选择题自动评分、总分、得分、正确题数、提交时间和成绩列表。

## 5. 当前范围

当前仓库包含：

- 根文档、AI 协作规则、产品事实文档、阶段总览、计划模板、安全说明、贡献说明和开源协议。
- 后端 Spring Boot 工程，包含统一响应、错误码、全局异常处理、健康检查、Spring Security、JWT、Flyway migration、用户权限基础表、菜单表、角色菜单关系和管理员种子账号。
- 后端管理接口包含用户管理、角色管理、权限清单、菜单清单、题库管理、试题管理、试卷管理、考试管理和成绩管理，并通过 `system:admin` 做方法级授权。
- 后端考试端接口包含考试任务、开始作答、提交试卷和我的成绩，并要求登录和考试权限。
- 前端 Vue 3 TypeScript 工程，包含登录页、管理端布局、考试端布局、路由守卫、API client、token store、用户管理、角色管理、权限清单、菜单清单、题库管理、试题管理、试卷管理、考试管理、成绩管理、考试任务列表、在线作答、题目附件展示、提交结果、403 和 404。
- Docker Compose 配置，描述 MySQL、Redis、后端和前端服务；本地 MySQL 使用 `13306`，Redis 使用 `16379`，避免和本机默认端口冲突。
- 根常规验证脚本，串联 Python 脚本编译、后端测试、前端类型检查、前端单元测试、前端构建、Docker Compose 配置检查和文档扫描，并严格处理命令失败。
- 根浏览器验收脚本，使用真实 Docker MySQL/Redis、真实 Spring Boot 后端、真实 Vue 前端和 Chromium，覆盖登录、管理端导航、角色保存、考试作答和提交结果。

## 6. 验收标准

- 根目录存在 `README.md`、`AGENTS.md`、`spec.md`、`plan.md`、`plan.example.md`、`LICENSE`、`SECURITY.md`、`CONTRIBUTING.md` 和 `.gitignore`。
- 根目录存在 `roadmap.md`，用于记录阶段总览和当前阶段。
- `.codex/skills/` 中存在开发、UI 和 plan 三个项目级 skill。
- 文档统一使用 kaoshi 作为项目名。
- 文档只记录当前确认的产品方向和工程规则。
- `backend/` 存在可测试的 Spring Boot 工程。
- `frontend/` 存在可类型检查和可构建的 Vue 3 TypeScript 工程。
- 管理员可以通过管理端维护用户、角色、权限关系和菜单关系。
- 管理员可以通过管理端维护题库、试题、试卷、考试和成绩。
- 登录用户可以通过考试端查看考试任务、开始作答、提交试卷并查看提交结果。
- 考试端可以展示题目图片、音频和文件链接附件。
- 系统可以对单选题和多选题自动评分，并归档成绩。
- 未登录请求不能访问管理接口。
- `start_test.py` 是当前常规验证入口，不打开浏览器。
- `start_browser_test.py` 是当前真实浏览器验收入口，会重置本项目 Docker 数据库并运行 Chromium 自动验收。
- `stop_test.py` 可以停止测试相关服务。
